# Pom Order — Design System

Generated by /plan-design-review on 2026-05-12.
Direction: **Utility-first** (Linear / Notion / Vercel style).

This file is the single source of truth for design tokens, component patterns, and interaction states. Companion to [`DESIGN-mvp.md`](DESIGN-mvp.md) (feature spec).

---

## Design Direction

**Utility-first.** Tool-feel. Beautiful through restraint, not decoration.

Reference projects: Linear (linear.app), Notion (notion.so), Vercel dashboard, Stripe Dashboard.

Heuristic: **if it doesn't help the user complete a task faster or with less doubt, cut it.**

## Typography

### Fonts

- **Display + body**: [Pretendard Variable](https://github.com/orioncactus/pretendard) — handles Korean (한글), Vietnamese (Tiếng Việt với dấu), and Latin in 1 file.
- **Monospace + numbers**: [Geist Mono](https://vercel.com/font/mono) — tabular numerics for money columns.
- **System fallback**: `system-ui, -apple-system, sans-serif` (only when Pretendard fails to load).

```css
:root {
  --font-sans: 'Pretendard Variable', system-ui, sans-serif;
  --font-mono: 'Geist Mono', ui-monospace, monospace;
}

/* Money columns */
.tabular { font-variant-numeric: tabular-nums; }
```

### Type scale (modular 1.25)

| Token | Size | Line-height | Use |
|---|---|---|---|
| `text-xs` | 12px | 16px (1.33) | Caption, label small |
| `text-sm` | 14px | 20px (1.43) | Secondary text, table cells |
| `text-base` | 16px | 24px (1.5) | Body default |
| `text-lg` | 20px | 28px (1.4) | Section heading |
| `text-xl` | 24px | 32px (1.33) | Page heading |
| `text-2xl` | 32px | 40px (1.25) | Big numbers (dashboard KPIs) |
| `text-3xl` | 40px | 48px (1.2) | Hero (rarely used in utility) |

**Vietnamese consideration**: with diacritics, increase line-height by 1.4× minimum. Pretendard handles this gracefully out of the box.

**Korean consideration**: Korean text is visually denser than Latin. Korean headings can use 0.95× font-size of Latin equivalent to balance visual weight.

### Weights

Pretendard Variable supports 100-900. Use:
- 400 (regular) — body
- 500 (medium) — table headers, secondary emphasis
- 600 (semibold) — UI labels, button text
- 700 (bold) — page headings, big numbers

Avoid 800/900 (too aggressive for utility).

## Spacing Scale

Tailwind default — 4px base unit, 8px grid.

| Token | Pixels | Use |
|---|---|---|
| `space-1` | 4px | Tight icon spacing |
| `space-2` | 8px | Compact components |
| `space-3` | 12px | Form field internal padding |
| `space-4` | 16px | Card content padding |
| `space-6` | 24px | Section gaps |
| `space-8` | 32px | Page section breaks |
| `space-12` | 48px | Major layout breaks |
| `space-16` | 64px | Hero spacing (rare in utility) |

**Density**: comfortable, not compact. Avoid rows < 40px height. Buttons ≥ 36px. Touch targets ≥ 44px on mobile.

---

## Radius

| Token | Pixels | Use |
|---|---|---|
| `radius-sm` | 4px | Badges, tags |
| `radius-md` | 6px | Buttons, inputs |
| `radius-lg` | 8px | Cards, modals |
| `radius-xl` | 12px | (rare — only for media containers) |
| `radius-full` | 9999px | Avatars, pill buttons |

**Anti-pattern**: don't apply same large radius to every element. That's the "bubbly AI slop" look. Distinct radius per element type signals craft.

---

## Motion

Restrained. Linear-style.

| Token | Duration | Easing | Use |
|---|---|---|---|
| `motion-fast` | 150ms | ease-out | Hover, focus, button press |
| `motion-base` | 250ms | ease-out | Modal open/close, drawer slide |
| `motion-slow` | 400ms | ease-in-out | Page transitions, status timeline animation |

**No spring animations.** No bouncing. No "playful" effects. Smooth, brief, purposeful.

**`prefers-reduced-motion`**: respect it — set all durations to 0.01ms when user prefers reduced.

## Component Patterns

### Buttons

Primary (CTA): `bg-accent text-accent-fg`, hover darker.
Secondary: `bg-surface-2 text-fg border border-border`, hover surface darker.
Destructive: `bg-danger text-white`, hover darker red.
Ghost (icon-only): no background, hover surface-2.

Always: `border-radius: 6px`, `padding: 8px 16px`, `font-weight: 600`, `font-size: 14px`.

### Inputs

Border: `1px solid var(--color-border)`. Focus: `2px solid var(--color-accent)` (outline, not border swap). Padding: `8px 12px`. Radius: 6px.

Vietnamese diacritics: ensure input field height ≥ 40px to avoid clipping.

### Tables

Header row: `bg-surface-2`, `text-xs`, `uppercase`, `text-fg-muted`, `font-weight: 600`.
Body rows: alternate `bg-surface` / `bg-bg`. Hover: `bg-surface-2`.
Numbers: right-aligned, `font-variant-numeric: tabular-nums`, `font-family: var(--font-mono)`.

### Cards

`bg-surface`, `border 1px solid border`, `radius-lg`, `padding 16px`. No box-shadow by default (Linear style). Use shadows ONLY for elevated states (modal, dropdown).

**Anti-AI-slop rule**: cards only when card IS the interaction (e.g., order card in a list). Not for decorative grouping.

### Status Timeline (Order Detail centerpiece)

This is the visual focal point of the order detail page. Vertical timeline với:
- Icon left (filled circle = done, outlined = pending, red X = problem)
- Connecting line (solid green = done, dashed grey = pending)
- Status name + timestamp + actor name

```
●─── Đã đặt (구매)         2026-05-10 14:30  bạn
│
●─── Đã ship từ Hàn         2026-05-12 09:15  (auto from Korean tracking)
│
○─── Về VN                   pending
│
○─── Giao khách              pending
│
○─── Tất toán                pending
```

## Information Architecture per Screen

### Dashboard
- **Primary**: tổng VND chờ thu tháng này (big number, text-2xl) + count đơn theo status (8 chip badges)
- **Secondary**: top brands tháng (3-5 items) + recent orders (5 rows table)
- **Tertiary**: quick actions floating (Tạo đơn / Set FX)
- **AI slop avoid**: NO 3-column "metric card" grid với colored circles + icons. Use a simple stat row with text-2xl numbers + supporting text-xs labels.

### Customer List
- **Primary**: search bar (top) + customer table
- **Secondary**: filter (active/all), sort, count
- **Tertiary**: "Tạo khách mới" button top-right

### Customer Detail
- **Primary**: tên + contact methods (Zalo, FB, phone)
- **Secondary**: lịch sử đơn (5 most recent, link to full list)
- **Tertiary**: edit, add address, add contact

### Order List
- **Primary**: filter chip bar (by status, all statuses visible) + table
- **Secondary**: search by customer/product, date range
- **Tertiary**: "Tạo đơn mới" button top-right, export to CSV

### Order Detail
- **Primary**: status timeline (centerpiece, vertical, 40% of viewport)
- **Secondary**: items + amounts + payment history
- **Tertiary**: customer info, "Share link" button (public token URL), edit, cancel

### New Order Flow (single-page form, not multi-step)
- Top: customer picker (autocomplete) → if not exists, inline "create new customer"
- Middle: items section (repeatable rows: brand + product name + URL + qty + KRW cost + VND sale price)
- Bottom: FX rate (auto-filled from current, editable), Korean shipping, international shipping, notes
- Right rail (desktop) or sticky bottom (mobile): live total + profit preview

### FX Rate Screen
- **Primary**: current rate (big number, text-2xl) + "Set new rate" button
- **Secondary**: history table (last 10 entries with effective_from/effective_to)
- **Tertiary**: source/notes field

### Payment Recording (modal, not page)
- **Primary**: amount + type (deposit/balance/refund) + method dropdown
- **Secondary**: paid_at (default now, editable), notes
- **Tertiary**: cancel, submit

### Public Order Page (`/o/{token}`)
- **Primary**: big status (current step + visual progress bar/timeline horizontal)
- **Secondary**: items list (read-only), amount summary, deposit status
- **Tertiary**: shop name + contact info (Zalo link, phone). NO customer PII (phone/address) leak.

## Interaction States (5-state pattern)

Every screen with data MUST specify all 5 states. From [`DESIGN-mvp.md`](DESIGN-mvp.md) Pass 2 review, locked patterns:

| State | Pattern |
|---|---|
| **Loading** | Skeleton placeholders matching final layout (no spinning circles for main content). Inline spinners only for actions (button loading). |
| **Empty** | Always include: friendly message (Vietnamese), primary CTA, illustration optional. NEVER "No results." |
| **Error** | Inline banner with retry button. Error message in Vietnamese. Log technical detail to console, not UI. |
| **Success** | Toast notification (auto-dismiss 3s) + UI update. Don't full-page reload. |
| **Partial** | Shimmer placeholder for slow sections. Don't block the whole screen. |

### Empty state copy guidelines

- ❌ "No items found"
- ✓ "Chưa có đơn nào. **Tạo đơn đầu tiên** để bắt đầu."

- ❌ "404 Not Found"
- ✓ "Đơn này không tồn tại hoặc đã bị xóa. Liên hệ shop để kiểm tra."

## Accessibility Baseline

- **Touch targets**: ≥ 44×44px on mobile (Apple HIG / Material), ≥ 36px on desktop
- **Color contrast**: 4.5:1 for body text, 3:1 for large text (WCAG AA)
- **Keyboard nav**: Tab order matches visual order. All interactive elements focusable. `:focus-visible` outline ≥ 2px.
- **ARIA landmarks**: `<header>`, `<nav>`, `<main>`, `<aside>`, `<footer>` on every page.
- **Screen reader**: form fields have `<label>` (visible OR `aria-label`). Status changes announced via `aria-live="polite"`.
- **Reduced motion**: respect `prefers-reduced-motion: reduce`.

## Anti-Patterns (AI Slop Blacklist)

Things to NOT do — every one of these screams "AI-generated template":

1. ❌ Purple-to-blue gradient backgrounds
2. ❌ 3-column feature grid (icon-in-circle + bold title + description)
3. ❌ Icons in colored circles as section decoration
4. ❌ Centered everything (text-align: center on all headings)
5. ❌ Same large border-radius on every element
6. ❌ Decorative blobs / floating circles / wavy SVG dividers
7. ❌ Emoji as design elements (🚀 in headings)
8. ❌ Colored left-border on cards
9. ❌ "Welcome to Pom Order" / "Your all-in-one solution for..." generic copy
10. ❌ system-ui as primary font (= "I gave up on typography")
