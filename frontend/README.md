# Pom Order Frontend

React 18 + Vite + TypeScript + Tailwind + shadcn/ui.

## Setup

```bash
npm install
cp .env.example .env  # fill in Supabase anon key
```

## Run

```bash
npm run dev          # http://localhost:5173
npm run build        # production build to dist/
npm run typecheck    # tsc --noEmit
npm test             # vitest
```

## Add shadcn/ui components

```bash
npx shadcn@latest add button
npx shadcn@latest add input dialog table
```

Components go to `src/components/ui/`.

## Structure

```
src/
├── main.tsx              Entry point
├── App.tsx               Top-level router
├── styles/globals.css    Design tokens (CSS variables) + Tailwind base
├── lib/
│   ├── utils.ts          cn() helper for class merging
│   └── api-client.ts     Typed fetch wrapper for backend
├── types/
│   ├── api.ts            Hand-typed for now
│   └── supabase.ts       Auto-generated by `make types-gen`
├── components/
│   └── ui/               shadcn/ui primitives (added on demand)
└── pages/                Route components (added in Tuần 3-4)
```

## Design tokens

All colors, spacing, typography defined as CSS variables in `src/styles/globals.css`.
Tailwind config (`tailwind.config.js`) maps those variables to utility classes.
See [`../docs/DESIGN.md`](../docs/DESIGN.md) for the design system rationale.

## Conventions

- Money display: `font-mono` + `tabular-nums` class, format VND with commas.
- Korean text: works out of the box with Pretendard.
- Status badge component: use `<Badge variant="ordered" />` (TBD when shadcn Badge installed).
